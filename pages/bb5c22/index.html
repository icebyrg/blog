# 列举数据类型检测方法

  • typeof [value]

    • 返回值是字符串，字符串中包含所属的数据类型

    • 特殊情况 typeof null → ‘object’

      • JS中创建的所有类型值，在计算机底层都是按照“二进制”进行存储的

      • typeof检测也是二进制值进行检测，其中就有一条规则：

        如果是以“000”开始的二进制值，则被识别为对象（null存储的二进制值都是0，符合以000开始）；然后再去看对象是否实现了[[call]]，实现了则为函数（返回“function”），没实现就是对象（返回“object”）

    • 因为是按照二进制值进行检测，所以检测速度是比较快的（最快）

    • typeof 检测对象类型，除函数被识别为字符串’function’，其余都是’object’【不能细分对象】

    • typeof 检测未被声明的变量，不会报错，结果是’undefined’

# Number()

规则：

  1. 字符串转换为数字：空字符串变为0，如果出现任何非有效数字，结果都是NaN
  2. 把布尔转换为数字：true→1 false→0
  3. null→0 undefined→NaN
  4. Symbol无法转换为数字，会报错：Uncaught TypeError: Cannot convert a Symbol value to a number
  5. BigInt 去除“n”（超过安全数字的，会按照科学计数法处理）
  6. 把对象转为数字，有四个步骤【类似的对象转为字符串 (opens new window)】
    1. 先调用对象的Symbol.toPrimitive这个方法，如果不存在这个方法
    2. 再调用对象的valueOf获取原始值，如果获取的值不是原始值
    3. 再调用对象的toString把其变为字符串
    4. 最后再把字符串基于Number方法转换为数字

示例：

  • 代码1

    let time = new Date();
console.log(Number(time));
/*
  首先检测Symbol.toPrimitive有没有，结果：有，而且是个函数
    time[Symbol.toPrimitive]('number')
*/

let arr = [10];
console.log(Number(arr));
/*
  首先 arr[Symbol.toPrimitive] -> undefined
  其次 arr.valueOf() 获取原始值 -> [10] 不是原始值
  再次 arr.toString() 转换为字符串 -> '10'
  最后 再把字符串'10'转换为数字 -> 10
*/

let num = new Number(10);
console.log(Number(num));
/*
  首先 num[Symbol.toPrimitive] -> undefined
  其次 num.valueOf() 获取原始值 -> 10 是原始值
*/

  • 代码2

[10][Symbol.toPrimitive]->undefined 不具备这个方法
[10].valueOf()->[10]不是原始值
[10].toString()->'10'
Number('10')->10
---

[10,20].toString()->'10,20'
Number('10,20')->NaN
---
let obj={0:10,length:1};
console.log(Number(obj)); //isNaN
obj[Symbol.toPrimitive] //undefined
obj.valueOf() //{0:10,length:1} 非原始值
obj.toString() //object原型的toString，用于检测数据类型->"[object Object]"->转为数字NaN
----

# parseInt()

console.log(parseInt("12px")); //12
// parseInt([value])或者parseFloat([value])
  • 一般用于手动转换
  • 规则：

    • [val] 值必须是一个字符串，如果不是则先转换为字符串

    • 然后从字符串左侧第一个字符开始找，把找到的有效数字字符最后转换为数字【一个都没找到就是NaN】

    • 遇到一个非有效数字字符，不论后面是否还有有效数字字符，都不再查找了

    • parseFloat可以多识别一个小数点

      parseInt([val], [radix])

在parseInt本身处理规则的基础上，还可以设置[radix]进制：在[val]字符串中，从左到右查找符合[radix]进制的内容（遇到一个不符合的则结束查找），把找到的内容当做[radix]进制值最后转换为10进制

  • 把其他进制转换为10进制 按权展开求和

    '10011'(2进制) → 10进制：

    12^4+02^3+02^2+12^1+1*2^0 → 16+0+0+2+1=19

    parseInt('10011',2)//19
parseInt('10011345',2) //19

  • [radix]不设置（或者设置为0），默认值是10，但是如果字符串是以0x开头的，则radix默认值是16

  • [radix]有取值范围 2~36，不在范围内的最后处理结果都是NaN

  • 在浏览器中有个特殊情况：如果数字（不是字符串）是以0开始的，浏览器会认为这应该是8进制的值，它会把8进制转换为10进制再处理

012->10
0*8^2+1*8^1+2*8^0 -> 0+8+2=10

语法

parseInt([val], [radix])

  • [val] 必须是字符串，如果不是，要先隐式转换为字符串 String([val])

  • [radix] 进制

    • 如果不写，或者写零：默认是10（特殊情况：如果字符串是以0x开始的，默认值是16）
    • 有效范围：2~36（计算机进制的范围），如果不在这个区间，结果直接是NaN

  • 从[val] 字符串左侧第一个字符开始查找，查找出符合[radix] 进制的值，遇到不符合的则结束查找，不论后面是否还有符合的；把找到的内容，按照[radix] 进制，转换为10进制

  • 用法示例

    console.log(parseInt('12px13', 2)); // 1
console.log(parseInt('10102px13', 2)); // 10
// 找到符合二进制的：'1010'
// 把这个二进制的值转换为10进制“按权展开求和”
// 1*2^3+0*2^2+1*2^1+0*2^0 -> 8+0+2+0

  • 字节题目

    let arr = [27.2, 0, '0013', '14px', 123];
arr = arr.map(parseInt);
console.log(arr);
/*
  parseInt(27.2, 0) -> parseInt('27.2', 10)
    '27' 把它当做10进制转换为10进制 -> 27
  parseInt(0, 1) -> 1 不符合进制规则
    'NaN'
  parseInt('0013', 2)
    '001' 把它当做2进制转换为10进制 -> 1
    0*2^2+0*2^1+1*2^0 -> 0+0+1 -> 1
  parseInt('14px', 3)
    '1' 把它当做3进制转换为10进制 -> 1
    1*3^0 -> 1
  parseInt(123, 4)
    '123' 把它当做4进制转换为10进制 -> 1
    1*4^2+2*4^1+3*4^0 -> 16+8+3 -> 27
*/

// map迭代数组中的每一项，传参当前项和索引，每次迭代都把返回值替换当前项
arr.map((item, index) => {
  // 27.2，0
  // 0，1
  // '0013'，2
  return item * 10;
});

  • 特殊情况：JS中以00开头的数字会被默认当成8进制转成10进制，之后才进行其他操作

    parseInt('0013', 2); // 1
parseInt(0013, 2); // 3
// 先8转10 0*8^3+0*8^2+1*8^1+3*8^0 -> 0+0+8+3 -> 11
// parseInt(11, 2);
//   '11'
//   再2转10 1*2^1+1*2^0 -> 2+1 -> 3

  • 使用parseInt([val], 10) 强制找10进制内容相对更加严谨

# Number类型的运算

0.1 + 0.2 !== 0.3

所有值在计算机低层都是按照二进制存储的，运算也基于二进制

  • 整数运算一般不会有问题
  • 浮点运算一般会出问题

十进制转二进制：

  • 整数：除2取余，最后倒着拼接，最后一位也要除
  • 小数：乘2取整，一直到结果是1为止
[(10进制)value].toString(2) // 转为2进制

浮点数转为2进制，会出现无限循环的情况，计算机底层存储的时候按照可识别的最长位数存储，其余的直接干掉，所以浮点数存储的二进制值本身就失去了精准度

所以最后运算的结果也是缺乏精准度的，而且小数最后面的0会省略，但凡不是0，都不会省略

# 浮点数运算

JS中有关小数（浮点数）的计算会出现精准度丢失的问题

  • JS中所有值都是以二进制在计算机低层进行存储的（浮点数转为二进制，可能出现无限循环的情况）
  • 在计算机低层存储的时候，最多存储64位【舍弃了一些值，值本身就失去了精准度】
  • 计算机设计问题

# 浮点数运算精度误差的解决方案

  1. toFixed保留小数点后面N位，它自己会四舍五入，返回值是字符串

    +(0.1+0.2).toFixed(2) // 0.3

  2. 扩大系数法

    • 运算保证精准度，实现思路：把小数变为整数（乘以系数）运算，运算后的结果再除以系数

      // 浮点数加减法运算，coefficient 系数
const coefficient = function coefficient(num) {
  num = num + "";
  let [, char = ""] = num.split("."),
    len = char.length;
  return Math.pow(10, len); // -> 10**len (ES6)
};
const plus = function plus(num1, num2) {
  num1 = +num1;
  num2 = +num2;
  if (isNaN(num1) || isNaN(num2)) return NaN;
  let max = Math.max(coefficient(num1), coefficient(num2));
  return (num1 * max + num2 * max) / max;
};
const minus = function minus(num1, num2) {
  num1 = +num1;
  num2 = +num2;
  if (isNaN(num1) || isNaN(num2)) return NaN;
  let max = Math.max(coefficient(num1), coefficient(num2));
  return (num1 * max - num2 * max) / max;
};

console.log(plus(0.2, 0.1)); // 0.3
console.log(minus(0.2, 0.1)); // 0.1
console.log(plus(0.1, 0.2534));

# 转字符串

  • 其他类型转为字符串 String([value])常用于隐式转换，或者[value].toString()常用于显式转换

    • String([value])如果value是个对象，转换有三个步骤【类似于对象转为数字 (opens new window)】

      • 先调用对象的Symbol.toPrimitive这个方法，如果不存在这个方法
      • 再调用对象的valueOf获取原始值，如果获取的值不是原始值
      • 再调用对象的toString把其变为字符串

    • String([obj]) 与[obj].toString是相等的吗？

      String([obj]) 要经过严格的三步骤，不一定会变成字符串

      [obj].toString 则是直接变成字符串，一定是变成字符串

      • 重点：对象转换为字符串基于这两种方式还是有区别的
      • 对象.toString() 直接调取所属类原型上的toString方法进行处理
      • String(对象)首先获取 对象[Symbol.toPrimitive]这个属性，如果存在这个属性，则返回对象[Symbol.toPrimitive]('string')
      • 如果不存在，则继续调用 对象.valueOf()，如果获取的是原始值则结束即可，如果不是，才会调用toString方法进行处理
      • 特殊点：标准普通对象调用的toString是Object.prototype.toString，是用来检测数据类型的，而不是转换为字符串的
  • 常见的情况：

    • 字符串拼接【+在JS中除了数学运算也有字符串拼接的意思】

      • 有两边，一边是字符串，则一定是字符串拼接

      • 有两边，一边是对象：可能会是字符串拼接，因为其要把对象转换为数字，转换过程中如果遇到转换为字符串，则直接变为字符串拼接；如果遇不到则还是数学运算

      • 只有一边 +[value]：一定是把[value]转换为数字的 [value] + ''⇒把value转换为字符串

        console.log(12 + [13]);
// [13][Symbol.toPrimitive] -> [13].valueOf()
//-> [13].toString() '13' 直接变为字符串拼接了 -> '1213'
console.log(12 + new Date());
// new Date()[Symbol.toPrimitive]('default') '时间字符串' -> 字符串拼接
console.log(12 + new Number(13));
// new Number(13)[Symbol.toPrimitive] -> new Number(13).valueOf() -> 13 -> 数学运算 -> 25

    • parseInt([value])如果[value]不是字符串也要先变成字符串再处理

    • alert/confirm/prompt/document.write 都会把内容变成字符串，然后再输出

# 装箱和拆箱

let num = 10;
console.log(num.toFixed(2));

num是原始值，不是对象，按常理来讲，是不能做“成员访问”的

默认装箱操作：通过new Number(num)变为非标准特殊对象，这样就可以调用toFixed了

let num = new Number(10);
console.log(num + 10);

在操作的过程中，浏览器会把num这个非标准特殊对象变为原始值【通过标准步骤】，这个操作叫做拆箱

装箱和拆箱都是浏览器默认数据类型隐式转换的过程

var i = 0;
Object.defineProperty(window, "a", {
  get() {
    return ++i;
  },
});
if (a == 1 && a == 2 && a == 3) {
  console.log("OK");
}
